# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| Latest  | Yes       |

Only the latest version of CityFlow receives security updates. If you are using an older version, please upgrade to the latest release.

## Reporting a Vulnerability

If you discover a security vulnerability in CityFlow, please report it responsibly. **Do not create a public GitHub issue for security vulnerabilities.**

### How to Report

Send an email to **security@sizops.co.il** with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### What to Include

- Vulnerability type (e.g., authentication bypass, injection, privilege escalation)
- Affected component (e.g., API endpoint, frontend page, backend service)
- Reproduction steps or proof of concept
- Logs, screenshots, or other relevant evidence

### What NOT to Report

- General bugs (use [GitHub Issues](../../issues))
- Feature requests (use [GitHub Issues](../../issues))
- UI issues or typos (use [GitHub Issues](../../issues))

## Response Process

1. **Acknowledgement** — We will acknowledge receipt of your report within 7 days
2. **Investigation** — We will investigate and assess the severity of the issue
3. **Fix** — We will develop and test a fix
4. **Release** — We will release a patched version
5. **Disclosure** — We will publicly disclose the issue after the fix is available

Critical vulnerabilities will be prioritized and handled as quickly as possible.

## Responsible Disclosure

We request that you:

- Allow us reasonable time to investigate and address the issue before public disclosure
- Provide reproduction steps so we can verify the vulnerability
- Avoid accessing or modifying data belonging to other users
- Report the issue in good faith and do not exploit it for malicious purposes

## Security Scope

The following are considered security-related issues:

- Authentication and authorization bypass
- Privilege escalation
- Sensitive data exposure
- Injection vulnerabilities (SQL, NoSQL, XSS, etc.)
- Broken access controls
- Session management issues
- API security problems
- Server or infrastructure misconfigurations
- Rate limiting bypass

## Out of Scope

- Feature requests
- General bugs not related to security
- UI or cosmetic issues
- Non-security suggestions

These should be reported through standard [GitHub Issues](../../issues).

## Acknowledgement

We appreciate the security research community and will acknowledge reporters who help improve CityFlow's security. Unless you prefer to remain anonymous, we will credit you in the release notes when a fix is published.
